Client Engagements

Queen City Scooters

DevSecOps Pipeline & AWS Security Hardening

Queen City Scooters is a small local scooter shop in Charlotte. They brought me in as a technical consultant. Small businesses get targeted because attackers know the security posture is usually weak. A compromised IAM key can rack up tens of thousands in AWS charges overnight from crypto mining alone, and for a shop that size that is a business-ending event. They had no CI/CD, no security scanning, and secrets sitting in plain text.

I built the entire DevSecOps pipeline from scratch. The GitHub Actions workflow covers the full delivery lifecycle: static code analysis and dependency scanning on every PR, container image builds with vulnerability scans before anything hits the registry, automated test runs, and gated deploys that stop cold if a scan fails. CloudWatch and SNS alarms watch Lambda functions and ECS containers in real time so the team knows within minutes if something goes wrong. I also wrote secret rotation scripts that cycle AWS credentials and RDS passwords on a schedule so stale exposed credentials never get the chance to become an attack vector.

A single compromised IAM key at a shop their size could mean a $30,000 or more AWS bill, customer data exposure, and PCI compliance violations from payment data. The cost of getting this right up front is a fraction of recovering from a breach. Once the pipeline was in place they also had the audit trail and security posture to qualify for a larger AWS build down the road without starting over.

GitHub Actions AWS ECS Lambda CloudWatch SNS Secret Rotation SAST Container Scanning Terraform IAM
Full DevSecOps pipeline from zero. Scans, builds, tests, gated deploys, real-time alarms, and automated secret rotation protecting a live business from credential compromise
Cortex

AI-Driven Incident Management Platform

Cortex came to Body Technology Solutions to build an MVP incident management platform from the ground up. Engineering teams were losing too much time manually digging through logs during incidents with no consistent process for capturing what happened or why.

I built a platform that automatically detects system failures, pulls the relevant logs, and runs them through an AI layer that generates plain-English incident summaries and postmortem data. Everything gets delivered directly to the team through pattern matching so they spend time fixing the problem instead of reconstructing what happened. The goal was to compress the gap between an incident starting and the team understanding it.

AI / LLM Incident Management Log Analysis Pattern Matching Postmortem Automation
MVP delivered. Faster incident response and less time spent manually digging through logs during outages
